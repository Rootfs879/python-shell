**Contributing Guidelines**

Thank you for your interest in contributing to this project! We welcome all contributions, whether they are bug reports, feature suggestions, or code improvements. By contributing, you help make this shell more powerful, efficient, and portable for everyone.

**How to Contribute**

1. Reporting Bugs
If you encounter any issues, please open a new issue in the [Issues](https://github.com/Rootfs879/python-shell/issues) section. Please provide as much detail as possible:
* **Description:** What happened?
* **Steps to Reproduce:** How can we replicate the issue?
* **Environment:** Are you running it on Desktop Linux or Termux (Android)? What is your Python version?
2. Requesting Features
We are always open to new ideas! Before starting any work, please start a conversation in [GitHub Discussions](https://github.com/Rootfs879/python-shell/discussions) so we can align on the implementation details.
3. Pull Requests (Code Improvements)

If you want to contribute directly to the code:
1. Fork this repository to your own GitHub account.
2. Create a new branch for your changes
3. Make your changes and ensure they follow the project's code style.
4. Test your changes locally (both on your PC and Termux if possible) to ensure everything works as expected.
5. Push your changes and submit a Pull Request to this repository.

**Coding Standards**

* Keep your code clean, readable, and well-documented.
* Add comments to complex parts of the logic.
* Ensure your code remains compatible with the current Python version used in the project.
